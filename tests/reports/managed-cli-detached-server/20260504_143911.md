## Executed Steps

1. Reviewed Windows output showing `ndxserver` starts the managed server and
   then incorrectly enters the interactive client login flow.
2. Identified that Windows npm shims do not preserve `ndxserver` in
   `process.argv[1]`, so binary-name detection falls back to normal `ndx` mode.
3. Added `src/cli/ndxserver.ts` bootstrap that sets
   `NDX_INVOKED_AS_SERVER=1` before importing the shared CLI main module.
4. Changed `package.json` so `bin.ndxserver` points to
   `dist/src/cli/ndxserver.js`.
5. Updated docs and managed startup test plan.
6. Ran `yarn exec prettier --write`.
7. Ran `yarn build`.
8. Ran `node --test dist/tests/cli-workspace.test.js`.
9. Ran `yarn test`.
10. Ran `npm run deploy`.
