# Test Report: managed-cli-detached-server

## Timestamp

2026-05-04 15:33:29 KST

## Scope

Investigated the Windows managed-server lifetime issue where a later `ndx`
invocation attempted to start a fresh server after the user exited a previously
working client session.

## Finding

The startup path was no longer the main failure: `ndxserver` could start a
background server and `ndx` could attach. The remaining lifetime risk was that
console signals from client exit could reach the managed server process. That
made the default server disappear even though it should have stayed alive until
an explicit server stop command.

## Change Verified

- Managed detached launches now pass `NDX_MANAGED_SERVER=1`.
- Server mode treats that environment variable as the background lifetime
  contract and ignores console `SIGINT`.
- Foreground `ndxserver serve` still exits on `SIGINT` or `SIGTERM`.
- `ndxserver stop` requests dashboard `/api/exit` and waits until the WebSocket
  endpoint is no longer reachable.
- Help output includes `ndxserver stop [--dashboard-listen HOST:PORT]`.
