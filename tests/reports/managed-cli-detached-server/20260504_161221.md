# Test Report: managed-cli-detached-server

## Timestamp

2026-05-04 16:12:21 KST

## Scope

Follow-up for the Windows managed-server lifetime issue after
`@neurondev/ndx@0.1.25` still restarted the server after a client exited at the
login prompt.

## Finding

The previous fix only ignored `SIGINT`. The user log showed the server was still
gone after the `ndx` client returned to PowerShell. The remaining likely failure
path is that Windows terminal shutdown can reach the detached Node process
through another terminal signal. The managed server therefore needs a stricter
background lifetime contract: terminal signals should not be the normal stop
path.

## Change Verified

- Managed server lifetime handling moved to `src/cli/lifetime.ts`.
- Managed servers ignore `SIGINT`, `SIGTERM`, `SIGHUP`, and `SIGBREAK`.
- Ignored signals are logged with guidance to use `ndxserver stop`.
- Foreground `ndxserver serve` still waits on `SIGINT` or `SIGTERM`.
- Managed server startup logs `managed lifetime active; stop with ndxserver
stop`.
