# Test Report: verdaccio-dashboard-login-release
## Timestamp
2026-05-05 12:52:12 KST

## Environment
- Repository: `ndx`
- Branch: `main`
- Registry: `https://verdaccio.neurondev.net/`
- Version: `0.1.27`
- Node package manager path: Yarn Plug'n'Play, npm publish to Verdaccio

## Executed Steps
1. Verified Verdaccio `@neurondev/ndx@0.1.26` was stale before release:
   `gitHead` pointed at `23be288...`, so it did not contain the merged local
   login prompt or managed server lifetime fixes.
2. Bumped package version and durable usage example to `0.1.27`.
3. Fixed dashboard API store queries to use the current `session` table and
   columns instead of removed `sessions/projects` tables.
4. Hardened dashboard HTTP error handling so render failures happen before
   headers are sent, and post-response request failures destroy the socket
   instead of writing a second HTTP response.
5. Ran `yarn install --immutable`.
6. Ran `yarn build` and verified built source has no
   `New Google login` or `Google login` prompt text.
7. Ran focused server tests with `node --test dist/tests/session-server.test.js`.
8. Ran full `yarn test`.
9. Ran `npm run deploy`.
10. Ran managed server signal verification with `NDX_MANAGED_SERVER=1`.
11. Ran `npm pack --dry-run --json`.
12. Published `@neurondev/ndx@0.1.27` to Verdaccio.
13. Verified Verdaccio metadata with `npm view`.
14. Installed `@neurondev/ndx@0.1.27` into `/tmp/ndx-install-verify-027` and
    verified binary versions and installed CLI prompt strings.
