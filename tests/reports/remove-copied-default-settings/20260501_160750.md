## Failures

- First service attempt exposed a regression: `tty: true` made `process.stdin`
  appear interactive, so `--mock` still fell through to the settings wizard.
  Fixed by allowing explicit `--mock` missing-settings fallback regardless of
  TTY state.
- Host-side `node dist/src/cli/main.js --mock --connect ...` failed in this
  environment because the host process could not create `/home/.ndx`. The
  socket path was verified inside the container where `/home/.ndx` is the
  expected runtime home.
