## Failures

- Initial session deletion test exposed `ERR_STREAM_WRITE_AFTER_END` when the
  server closed a socket before the JSON-RPC response write. Fixed by making
  WebSocket sends no-op after socket end.
- The same path then surfaced the expected client-side close rejection; the
  test now accepts the close after verifying `session/deleted`.
- `npm run deploy` did not reach in-container tests or mock-agent verification
  because the deploy Dockerfile clones
  `https://github.com/hikaMaeng/ndx.git@codex/restorecontext`, and that branch
  is not present on the remote.

## Reproducibility

- Re-run with `npm test`.
- Re-run deploy after pushing `codex/restorecontext` to the configured remote,
  or override `NDX_GIT_REF` to a remote branch containing this commit.
- The deletion integration test creates two temporary session servers, deletes a
  persisted session from one, and verifies the other emits `session/deleted` on
  the next prompt attempt.
